src/db/schema.ts:
- Export CATEGORIES as a readonly const array with exactly 14 strings in this order: 'Nouns', 'Verbs', 'Adjectives', 'People', 'Food', 'Animals', 'Vehicles', 'Body Parts', 'Onomatopoeia', 'Requests', 'Social Communication', 'Emotions', 'Places', 'Other'

- Export interface ChildProfile with id?: number, name: string, birthDate: string, languages: string[], createdAt: string, prematureBirth?: boolean, speechTherapy?: boolean, neurologicalCare?: boolean, parentNotes?: string — all 4 optional Phase 2 fields included in v1 per D-08
- Export interface WordForm with id?: number, form: string, createdAt: string
- Export interface Meaning with id?: number, text: string, categories: Category[], isActive: boolean, firstUseDate: string, lastUseDate: string — categories is Category[] per D-09, NOT a junction table
- Export interface WordFormMeaning with id?: number, wordFormId: number, meaningId: number — exactly D-11 fields, no extra metadata

src/db/db.ts:
- Import Dexie and EntityTable from 'dexie'
- Import entity interfaces from './schema'
- Define class AppDB extends Dexie with 4 EntityTable properties using ! assertion
- In constructor: call super('LittleWordsDB'), then this.version(1).stores() with:
  - childProfile: '++id'
  - wordForms: '++id, form, createdAt'
  - meanings: '++id, isActive, firstUseDate, lastUseDate, *categories' — *categories is the multi-entry index per D-09
  - wordFormMeanings: '++id, wordFormId, meaningId, [wordFormId+meaningId]' — compound index per D-11
- Export: export const db = new AppDB()
- All entities use ++id auto-increment per D-12

src/db/types.ts:
- Re-export everything from './schema': all 4 interfaces, Category type, CATEGORIES const

Service stubs (create directory src/db/services/ first):

src/db/services/childProfile.service.ts:
- Import db from '../db', ChildProfile from '../types'
- Export async function saveChildProfile(profile: Omit<ChildProfile, 'id'>): Promise<number> — body: return db.childProfile.add(profile)
- Export async function getChildProfile(): Promise<ChildProfile | undefined> — body: return db.childProfile.toCollection().first()

src/db/services/wordForm.service.ts:
- Import db from '../db', WordForm from '../types'
- Export async function addWordForm(form: Omit<WordForm, 'id'>): Promise<number> — body: return db.wordForms.add(form)
- Export async function deleteWordForm(id: number): Promise<void> — MUST wrap in db.transaction('rw', [db.wordForms, db.wordFormMeanings], async () => { await db.wordForms.delete(id); await db.wordFormMeanings.where('wordFormId').equals(id).delete(); }) — this is the canonical Pitfall 5 prevention pattern

src/db/services/meaning.service.ts:
- Import db from '../db', Meaning from '../types'
- Export async function addMeaning(meaning: Omit<Meaning, 'id'>): Promise<number> — body: return db.meanings.add(meaning)
- Export async function toggleMeaningActive(id: number, isActive: boolean): Promise<void> — body: await db.meanings.update(id, { isActive })

src/db/services/wordFormMeaning.service.ts:
- Import db from '../db', WordFormMeaning from '../types'
- Export async function linkMeaning(wordFormId: number, meaningId: number): Promise<number> — body: return db.wordFormMeanings.add({ wordFormId, meaningId })
- Export async function unlinkMeaning(wordFormId: number, meaningId: number): Promise<void> — body: await db.wordFormMeanings.where('[wordFormId+meaningId]').equals([wordFormId, meaningId]).delete()
