This task initializes the project from scratch. The working directory is the project root (the git repo root, where package.json will live).

STEP 1 — Scaffold with create-vite (if package.json does not already exist):
Run: npm create vite@7 . -- --template react-ts
Accept any prompts. The dot (.) scaffolds into the current directory.

PITFALL: If create-vite prompts "Current directory is not empty — continue?", select "Ignore files and continue" so the .planning/ .claude/ directories are preserved.

STEP 2 — Install runtime dependencies:
Run the following commands in sequence:
  npm install react-router@^7
  npm install dexie@4.4.4 dexie-react-hooks
  npm install i18next@^23 react-i18next@^15
  npm install zustand@5.0.14

STEP 3 — Install dev and build dependencies:
  npm install -D @tailwindcss/vite
  npm install -D vite-plugin-pwa@1.3.0
  npm install -D vitest @testing-library/react @testing-library/dom @testing-library/user-event jsdom

PITFALL (CRITICAL): Do NOT install @vitejs/plugin-react separately. The create-vite@7 template installs @vitejs/plugin-react@5.x automatically. Running npm install -D @vitejs/plugin-react without a version pin installs v6.x, which requires Vite 8 and will break the build.

STEP 4 — Write vite.config.ts (overwrite the create-vite default):
File: vite.config.ts
Content:
  - Import: defineConfig from 'vite', react from '@vitejs/plugin-react', tailwindcss from '@tailwindcss/vite', VitePWA from 'vite-plugin-pwa'
  - base: '/little-words/' — this is IMMUTABLE and MUST be set exactly. It must match the GitHub repository slug.
  - plugins: [react(), tailwindcss(), VitePWA({ ... })]
  - VitePWA config:
    - registerType: 'autoUpdate'
    - injectRegister: 'auto'
    - workbox: { globPatterns: ['**/*'], navigateFallback: null } — navigateFallback MUST be null for hash routing; do not set it to 'index.html'
    - includeAssets: ['**/*']
    - manifest: { name: 'Little Words', short_name: 'LittleWords', description: "Track your child's vocabulary development", theme_color: '#ffffff', background_color: '#ffffff', display: 'standalone', start_url: '/little-words/#/', scope: '/little-words/', icons: [] }
    - Icons array is intentionally empty — PWA icons are added in Phase 5

STEP 5 — Write tsconfig.app.json (overwrite the create-vite default):
File: tsconfig.app.json
Required settings:
  - target: "ES2020"
  - useDefineForClassFields: true
  - lib: ["ES2020", "DOM", "DOM.Iterable"]
  - module: "ESNext"
  - skipLibCheck: true
  - moduleResolution: "Bundler"
  - allowImportingTsExtensions: true
  - isolatedModules: true
  - moduleDetection: "force"
  - noEmit: true
  - jsx: "react-jsx"
  - strict: true
  - noUnusedLocals: true
  - noUnusedParameters: true
  - noFallthroughCasesInSwitch: true
  - noUncheckedSideEffectImports: true
  - baseUrl: "."
  - paths: { "@/*": ["./src/*"] }
  - types: ["vite/client", "vite-plugin-pwa/client"] — CRITICAL: without vite-plugin-pwa/client, TypeScript cannot resolve virtual:pwa-register imports in Phase 5
  - include: ["src"]

STEP 6 — Update index.html:
Set the title to "Little Words". Ensure the script src references './src/main.tsx' (relative, not absolute — Vite handles the base path injection at build time).

STEP 7 — Delete the create-vite boilerplate src/ files that will be replaced:
Delete: src/App.css, src/assets/react.svg, public/vite.svg (if they exist). Do not delete src/App.tsx or src/main.tsx yet — they will be overwritten in Wave 4.
