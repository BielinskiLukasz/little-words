STEP 1 — Write src/shared/components/ErrorBoundary.tsx (D-22, D-23):
- Class component extending Component<Props, State>
- Props: { children: ReactNode }
- State: { hasError: boolean; error: Error | null }
- Constructor: super(props), this.state = { hasError: false, error: null }
- Static getDerivedStateFromError(error: Error): State — returns { hasError: true, error }
- componentDidCatch(error: Error, info: ErrorInfo) — console.error only (no analytics per privacy constraint)
- render(): if hasError, show fallback div with:
  - h1: "Little Words"
  - if import.meta.env.DEV: <pre> showing this.state.error?.message (MUST use import.meta.env.DEV — NOT process.env.NODE_ENV)
  - else: <p> "Something went wrong" (D-23 exact text)
  - A button: onClick={() => window.location.reload()}, text "Reload"
  - Use CSS classes: bg-background, text-foreground, text-destructive, bg-muted for the pre block
- else: return this.props.children

STEP 2 — Write src/App.tsx (D-16, D-17, D-18):
Use the "simpler version" from RESEARCH.md — render OnboardingPage directly, not via a second RouterProvider:

Import RouterProvider from 'react-router' (check if it needs 'react-router/dom' — see Open Question A4 in RESEARCH.md)
Import useLiveQuery from 'dexie-react-hooks'
Import useTranslation from 'react-i18next'
Import db from './db/db'
Import router from './router'
Import ErrorBoundary from './shared/components/ErrorBoundary'
Import OnboardingPage from './pages/OnboardingPage'

Define function AppGate():
  - const { t } = useTranslation('common')
  - const profileCount = useLiveQuery(() => db.childProfile.count())
  - if profileCount === undefined: render splash div:
    <div className="flex h-[100dvh] items-center justify-center bg-background">
      <h1 className="text-2xl font-bold text-primary">{t('app.name')}</h1>
    </div>
  - if profileCount === 0: return <OnboardingPage />
  - return <RouterProvider router={router} />

Define and export default function App():
  return <ErrorBoundary><AppGate /></ErrorBoundary>

STEP 3 — Write src/main.tsx (import order is CRITICAL):
The import order must be:
  1. import './i18n/index';        // FIRST: i18n must init before React tree mounts
  2. import React from 'react';
  3. import ReactDOM from 'react-dom/client';
  4. import App from './App';
  5. import './index.css';         // Tailwind + Shadcn CSS variables

Then: ReactDOM.createRoot(document.getElementById('root')!).render(<React.StrictMode><App /></React.StrictMode>)

PITFALL: If CSS is imported before i18n, there is no functional breakage, but the i18n import must be first to guarantee synchronous initialization before any React component renders.
