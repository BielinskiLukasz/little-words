<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user text input → MeaningAutocomplete query | User-typed prefix passed to useLiveQuery via Dexie where() |
| user text input → addWordEntry | Form values passed to service before IndexedDB write |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation |
|-----------|----------|-----------|----------|-------------|------------|
| T-02-04-T1 | Tampering | word form text input | low | mitigate | Text is plain string stored in IndexedDB; React escapes all JSX output; no HTML injection possible through standard JSX rendering |
| T-02-04-T2 | Tampering | MeaningAutocomplete suggestion injection | low | mitigate | Suggestions come from the app's own IndexedDB (device-local data); no external data source; React auto-escapes all rendered text |
| T-02-04-I1 | Information Disclosure | category selection state in memory | low | accept | Categories are persisted in IndexedDB on save only; ephemeral state clears when sheet closes; no sensitive data |
| T-02-04-D1 | Denial of Service | useMeaningSearch without limit | low | mitigate | .limit(10) cap on Dexie query prevents large result sets blocking the render thread (per Pitfall 5 in RESEARCH.md) |
</threat_model>
